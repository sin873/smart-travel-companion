# 智能伴旅小程序 - 6小时激进并行开发方案

## ⚡ 开发策略总览

**开发时间**: 6小时
**开发模式**: 多窗口Qwen Code CLI并行敏捷开发
**分支策略**: 4个并行分支 + 1个集成分支

## 📋 接口契约（先行定义！）

### 文件位置
`docs/travel-api-contract.md`

### 统一响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "timestamp": 1234567890
}
```

### 1. 生成行程
**POST** `/api/v1/travel/plan`

**Request**:
```json
{
  "destination": "北京",
  "startDate": "2024-05-01",
  "endDate": "2024-05-03",
  "travelerCount": 2,
  "budget": 2000,
  "preferences": {
    "pace": "moderate",
    "interests": ["historical", "food"]
  }
}
```

**Response**:
```json
{
  "taskId": "task_abc123",
  "status": "QUEUED",
  "estimatedTime": 15
}
```

### 2. 查询规划状态
**GET** `/api/v1/travel/plan/{taskId}`

**Response**:
```json
{
  "taskId": "task_abc123",
  "status": "PROCESSING",
  "progress": 65,
  "message": "正在规划路线...",
  "itinerary": null
}
```

status枚举: `QUEUED`, `PROCESSING`, `COMPLETED`, `FAILED`

### 3. 保存行程
**POST** `/api/v1/travel/itineraries`

**Request**: 完整的ItineraryDTO

**Response**:
```json
{
  "itineraryId": "itin_xyz789"
}
```

### 4. 行程列表
**GET** `/api/v1/travel/itineraries`

**Response**:
```json
{
  "total": 5,
  "items": [
    {
      "itineraryId": "itin_xyz789",
      "title": "北京3日游",
      "destination": "北京",
      "startDate": "2024-05-01",
      "endDate": "2024-05-03",
      "status": "PLANNED"
    }
  ]
}
```

### 5. 行程详情
**GET** `/api/v1/travel/itineraries/{id}`

**Response**: 完整的ItineraryDTO

### 6. 删除行程
**DELETE** `/api/v1/travel/itineraries/{id}`

**Response**: `{ "code": 200, "message": "success" }`

## ✅ 验收标准（6小时后）

### 必须演示成功的流程

1. **用户打开小程序**
   - 进入行程定制页面
   - 填写：目的地（北京）、日期（5.1-5.3）、人数（2）
   - 点击"开始规划"

2. **系统生成行程**
   - 显示规划进度（0% → 100%，约5-10秒）
   - 进度条 + 步骤指示器
   - 显示"正在分析需求" → "正在检索景点" → "正在规划路线" → "生成完成"

3. **查看生成的行程**
   - 显示3天的行程
   - 每天2-3个景点
   - 每个景点有名称、时间、简介

4. **保存行程**
   - 点击"保存"按钮
   - 提示保存成功

5. **我的行程**
   - 查看行程列表
   - 刚才保存的行程在列表中
   - 点击查看详情
   - 可以删除行程

### 代码质量要求
- ✅ 能编译通过
- ✅ 不影响现有功能
- ✅ 有基本的异常处理
- ✅ 有日志输出
- ❌ 不需要完整的单元测试（演示为主）
