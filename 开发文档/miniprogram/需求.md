# 旅游规划模块最小闭环需求说明

## 一、核心目标

在现有派聪明项目基础上，新增一个最小可用的旅游规划小程序模块，通过AI能力为用户生成个性化旅游行程并支持基本的行程管理。

---

## 二、最小业务闭环

本期实现以下完整业务流程：

1. **用户填写旅游需求表单** - 输入目的地、时间等基本信息
2. **系统生成行程** - 调用AI能力生成完整行程方案
3. **查看生成的行程** - 展示行程详情和每日安排
4. **保存行程** - 将满意的行程保存到数据库
5. **查看我的行程列表** - 管理已保存的行程
6. **查看行程详情** - 查看某个行程的完整信息
7. **重新生成行程** - 基于相同或修改后的需求重新生成

---

## 三、用户输入项

### 本期必须保留的字段（最小集）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 目的地 | String | 是 | 如"北京"、"上海" |
| 开始日期 | Date | 是 | 行程开始日期 |
| 结束日期 | Date | 是 | 行程结束日期（自动计算天数） |
| 出行人数 | Integer | 是 | 默认2人 |

### 本期建议包含的增强字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 预算范围 | Decimal | 否 | 可选，如500-1000元/天 |
| 兴趣偏好 | List[String] | 否 | 多选：历史文化、自然风光、美食、购物等 |
| 出行节奏 | String | 否 | 单选：休闲、适中、紧凑 |

### 明确放到后续迭代的字段

- 出发地（需要交通规划能力）
- 出行方式（飞机/高铁/自驾）
- 同行类型（亲子/情侣/朋友/独自）
- 特殊要求（无障碍/素食/带宠物等）
- 酒店偏好（星级/价位/位置）
- 具体景点偏好（指定必去景点）
- 避开景点（指定不去的景点）

### 本期必须输出的核心内容

| 输出项 | 说明 |
|--------|------|

### 明确放到后续迭代的输出

- 实时天气预报
- 详细交通路线（公交/地铁具体线路）
- 预计花费明细（门票/餐饮/交通）

- 酒店推荐
- 人流量预测

- 最佳拍照位置推荐

## 五、本期明确不做的功能

以下功能本期坚决不做：

1. **预订相关**
   - 酒店预订
   - 门票预订
   - 机票/火车票预订
   - 租车预订

2. **支付相关**
   - 任何支付功能
   - 订单管理
   - 退款处理

3. **社交相关**
   - 行程分享给好友
   - 行程公开/私密设置
   - 点赞/评论/收藏
   - 关注其他用户

4. **复杂交互**
   - 拖拽调整行程顺序
   - 手动增删景点
   - 多人协同编辑
   - 行程模板市场

5. **高级算法**
   - 复杂的个性化推荐算法
   - 实时路况集成
   - 动态重规划
   - 多目标优化（最短路径/最少花费/最佳体验）

6. **内容社区**
   - 用户发表游记
   - 攻略UGC
   - 景点评分评价

7. **账号体系扩展**
   - 手机号绑定
   - 第三方登录（非微信）
   - 用户等级/积分体系

## 六、外部依赖边界

### 高德MCP负责什么

- 地点查询：根据名称获取景点详细信息
- 地理编码：地址转经纬度
- 路线规划（简化）：两点之间的大概距离和交通方式
- POI搜索：目的地周边景点、餐饮等

### 大模型负责什么

- 行程整体规划：根据用户需求编排景点顺序
- 景点描述生成：为每个景点生成推荐理由和介绍
- 行程文案润色：生成吸引人的行程标题和摘要
- 时间分配建议：合理分配每个景点的游览时间

### 本项目本地后端负责什么

- 用户认证（复用现有JWT体系）
- 需求数据校验和存储
- 行程数据的CRUD操作
- 调用高德MCP和大模型的编排
- 进度状态管理（生成中/完成/失败）
- API接口提供

---

## 七、建议的最小接口集合

从需求角度，本期只需以下接口：

| 接口名称 | 方法 | 路径 | 说明 |
|---------|------|------|------|
| 生成行程 | POST | `/api/v1/travel/plan` | 提交需求，触发行程生成，返回任务ID |
| 查询生成状态 | GET | `/api/v1/travel/plan/{taskId}` | 查询行程生成进度和结果 |
| 保存行程 | POST | `/api/v1/travel/itineraries` | 将生成的行程保存到我的行程 |
| 查询行程列表 | GET | `/api/v1/travel/itineraries` | 获取我的行程列表 |
| 查询行程详情 | GET | `/api/v1/travel/itineraries/{id}` | 获取单个行程的完整详情 |
| 删除行程 | DELETE | `/api/v1/travel/itineraries/{id}` | 删除某个行程 |

**说明**：不需要"重新生成"的独立接口，重新生成就是再次调用"生成行程"接口。

## 八、建议的最小数据对象

从需求角度，本期只需以下核心对象：

| 对象名称 | 核心属性 | 说明 |
|---------|---------|------|
| 旅游需求 | 目的地、开始日期、结束日期、人数、预算、偏好 | 用户提交的原始需求 |
| 行程主表 | 行程ID、用户ID、标题、目的地、日期范围、状态 | 行程的基本信息 |
| 每日行程 | 所属行程ID、第几天、日期、当日主题 | 按天的分组 |
| 行程明细 | 所属每日行程ID、景点名称、位置、时间、描述 | 具体的每个景点/活动 |
| 生成任务 | 任务ID、用户ID、状态、进度、请求数据、结果数据 | 异步生成过程的状态追踪 |

## 需求边界结论

### 🎯 这期一定做什么

1. **用户需求收集**
   - 目的地、日期、人数3个必填项
   - 预算、偏好、节奏3个可选项

2. **AI行程生成**
   - 调用大模型+高德MCP生成行程
   - 异步任务+进度查询
   - 基础的行程方案（景点+时间+描述）

3. **行程管理**
   - 保存生成的行程
   - 我的行程列表
   - 行程详情查看
   - 删除行程

4. **小程序前端**
   - 简洁的需求表单页
   - 生成进度展示页
   - 行程详情展示页
   - 我的行程列表页

5. **后端API**
   - 6个核心接口（生成、查状态、保存、列表、详情、删除）

### ❌ 这期一定不做什么

- 任何预订功能（酒店、门票、交通）
- 任何支付功能
- 社交分享、评论收藏
- 拖拽编辑、手动调整
- 多人协作
- 复杂推荐算法
- 实时路况、动态重规划

### 📅 建议放到后续迭代

**v1.1 优化体验**
- 拖拽调整行程顺序
- 手动增删景点
- 出发地+交通规划
- 更多筛选条件

**v1.2 丰富内容**
- 天气预报
- 详细花费预算
- 周边推荐
- 人流量提示

**v1.3 社交功能**
- 行程分享
- 收藏点赞
- 精选行程推荐
- 用户评价

## 与现有派聪明项目的集成原则

1. **复用现有能力**
   - 用户认证体系（JWT）
   - 数据库连接和事务管理
   - 异常处理机制
   - 日志记录
   - 配置管理

2. **最小改动原则**
   - 不修改现有代码
   - 只在对应目录下新增旅游规划相关类
   - 遵循现有的命名规范和代码风格

3. **独立模块**
   - Controller: `controller/travel/`
   - Service: `service/travel/`
   - Entity: `entity/travel/`
   - Repository: `repository/travel/`
   - Model: `model/travel/`
