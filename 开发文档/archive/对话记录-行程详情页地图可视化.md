
# 智能伴旅项目 - 行程详情页地图可视化功能开发记录

## 项目背景
完善智能伴旅（Smart Travel Companion）微信小程序 + Spring Boot 后端项目，实现旅游规划、保存、查看、删除功能，并接入高德地图 POI 搜索获取真实景点数据。

## 功能需求
在行程详情页增加地图可视化功能，展示所有景点位置（marker），自动居中地图，仅做打点展示（最小实现）。

---

## 开发过程

### 阶段一：任务理解与初始规划
1. 目标：在 `pages/itinerary/detail/index` 页面增加地图组件
2. 功能要求：
   - 展示所有景点位置 marker
   - 自动居中地图
   - 从 itinerary.days[].items[] 提取坐标

### 阶段二：代码实现（遇到编码问题）


### 阶段三：解决方案 - 使用 Git 恢复
1. 使用 Git 恢复原始文件：
   ```bash
   git restore miniprogram/pages/itinerary/itinerary.js
   git restore miniprogram/pages/itinerary/itinerary.wxml
   ```

2. 分析原始代码结构：
   - 原始文件使用 `day.attractions` 字段（而非 `day.items`）
   - 使用 ES5 语法（var、for 循环）兼容性更好

### 阶段四：最终实现（小心修改）

#### 1. itinerary.js 修改
```javascript
// data 字段新增
data: {
  itinerary: null,
  from: '',
  dateRange: '',
  markers: [],              // 新增
  centerLatitude: 0,        // 新增
  centerLongitude: 0        // 新增
}

// processItinerary 方法后新增调用
this.setData({ itinerary })
this.generateMapMarkers(itinerary)

// 新增 generateMapMarkers 方法
generateMapMarkers: function(itinerary) {
  var markers = []
  var markerIndex = 0

  if (itinerary.days &amp;&amp; itinerary.days.length &gt; 0) {
    for (var i = 0; i &lt; itinerary.days.length; i++) {
      var day = itinerary.days[i]
      var items = day.items || day.attractions || []  // 兼容两种字段
      if (items.length &gt; 0) {
        for (var j = 0; j &lt; items.length; j++) {
          var item = items[j]
          if (item.latitude &amp;&amp; item.longitude) {
            var lat = item.latitude
            var lng = item.longitude

            if (typeof lat === 'string') {
              lat = parseFloat(lat)
            }
            if (typeof lng === 'string') {
              lng = parseFloat(lng)
            }

            if (!isNaN(lat) &amp;&amp; !isNaN(lng)) {
              markers.push({
                id: markerIndex++,
                latitude: lat,
                longitude: lng,
                title: item.attractionName || item.name || '景点',
                width: 30,
                height: 30
              })
            }
          }
        }
      }
    }
  }

  var centerLat = 0
  var centerLng = 0
  if (markers.length &gt; 0) {
    centerLat = markers[0].latitude
    centerLng = markers[0].longitude
  }

  this.setData({
    markers: markers,
    centerLatitude: centerLat,
    centerLongitude: centerLng
  })
}
```

## 关键经验总结

1. **Git 的重要性**：在修改代码前确保有 Git 备份，出现问题可快速恢复
2. **编码问题**：Windows 环境下文件编码可能出现乱码，需小心处理
3. **兼容性优先**：使用 ES5 语法（var、for 循环）比 ES6 箭头函数兼容性更好
4. **字段兼容**：实际项目中数据结构可能有多种变体（items vs attractions），需做好兼容
5. **渐进式开发**：先恢复原始文件，再小心地、逐步地添加新功能
