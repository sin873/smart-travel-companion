## 1. 测试概述

### 1.1 测试目标
验证旅游规划模块后端 API 骨架的 6 个核心接口功能正确性，确保接口层可联调。

### 1.2 测试范围
| 接口 | 测试内容 |
|------|----------|
| POST `/api/v1/travel/plan` | 生成行程（创建任务） |
| GET `/api/v1/travel/plan/{taskId}` | 查询规划状态 |
| POST `/api/v1/travel/itineraries` | 保存行程 |
| GET `/api/v1/travel/itineraries` | 查询行程列表 |
| GET `/api/v1/travel/itineraries/{itineraryId}` | 查询行程详情 |
| DELETE `/api/v1/travel/itineraries/{itineraryId}` | 删除行程 |

### 1.3 不测试范围
- ❌ 高德 MCP 集成
- ❌ DeepSeek / LLM 集成
- ❌ WebSocket 实时推送
- ❌ 真实行程生成算法

## 4. 详细测试用例

### 用例 1：生成行程

| 项目 | 内容 |
|------|------|
| **用例编号** | TC-001 |
| **接口名称** | 生成行程 |
| **请求方法** | POST |
| **请求路径** | `/api/v1/travel/plan` |

#### 请求 Headers
| 参数名 | 参数值 | 必填 |
|--------|--------|------|
| Content-Type | application/json | 是 |
| Authorization | Bearer &lt;token&gt; | 否 |

#### 请求 Body
```json
{
  "destination": "北京",
  "startDate": "2024-05-01",
  "endDate": "2024-05-03",
  "travelerCount": 2,
  "budget": 2000,
  "preferences": {
    "pace": "moderate",
    "interests": ["historical", "food"],
    "avoidCrowds": false
  }
}
```

#### 预期响应
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "taskId": "task_xxxxxxxx",
    "status": "PROCESSING",
    "estimatedTime": 15
  },
  "timestamp": 1714567890123
}
```

#### 验证点
- [ ] HTTP 状态码为 200
- [ ] `code` 为 200
- [ ] `data.taskId` 非空，格式为 `task_` 开头
- [ ] `data.status` 为 "PROCESSING"
- [ ] 保存返回的 `taskId` 供后续测试使用

## 5. 测试流程

### 5.1 完整测试流程

```
1. 启动后端服务
   ↓
2. TC-001: 生成行程 → 保存 taskId
   ↓
3. TC-002: 必填字段校验
   ↓
4. TC-003: 查询规划状态（使用刚才的 taskId）
   ↓
5. TC-004: 保存行程 → 保存 itineraryId
   ↓
6. TC-005: 查询行程列表 → 确认行程已保存
   ↓
7. TC-006: 查询行程详情（使用刚才的 itineraryId）
   ↓
8. TC-004: 再保存一个行程（用于测试删除）
   ↓
9. TC-005: 确认列表有 2 个行程
   ↓
10. TC-007: 删除其中一个行程
    ↓
11. TC-005: 确认列表只剩 1 个行程
```

### 5.2 数据准备建议

| 操作 | 说明 |
|------|------|
| 保存行程 1 | 用于测试详情和列表 |
| 保存行程 2 | 用于测试删除功能 |

## 6. 注意事项

### 6.1 数据格式要求

| 字段类型 | 格式要求 | 示例 |
|----------|----------|------|
| 日期 | `YYYY-MM-DD` | `2024-05-01` |
| 时间 | `HH:mm:ss` | `09:00:00` |
| 经纬度 | 十进制，7位小数 | `39.9163000` |

### 6.2 占位实现说明

| 接口 | 占位内容 |
|------|----------|
| 生成行程 | 只创建任务记录，不真实生成行程 |
| 查询规划状态 | 状态固定为 PROCESSING，itinerary 为 null |

### 6.3 用户认证

- **有 Token**：从 Authorization header 提取 userId
- **无 Token**：使用演示用户 `demo_user_001`
